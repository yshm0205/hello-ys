# 🛡️ 보안 및 성능 최적화 (권장 가이드)

FireShip은 서비스의 안전한 운영을 위해 엔터프라이즈급 보안 가이드를 제공합니다. 초보자도 이해하기 쉬운 비유와 함께 각 보안 항목을 확인해 보세요.

## 💡 쉽게 이해하는 보안 비유 (Metaphors)

컴퓨터 언어는 어렵지만, 우리 일상에 비유하면 아주 쉽습니다.

- **🚦 API 속도 제한 (Rate Limiting)**:
  - **비유**: "회전문에 1분에 5명만 들어가기"
  - **설명**: 카페에 한 사람이 숨 가쁘게 1초에 10번씩 들어왔다 나갔다 하면 영업이 방해되겠죠? 속도 제한은 그런 '장난'이나 '공격'을 하는 사람에게 잠시 밖에서 숨을 고르고 오라고 말하는 친절한 관리자입니다.
- **🔒 보안 헤더 (Security Headers)**:
  - **비유**: "이중 잠금장치와 신분증 검사"
  - **설명**: 내 집(웹사이트)을 누군가 가짜 상자로 감싸서 사기를 치거나(클릭재킹), 수상한 물건을 몰래 들여오는 것을 입구에서부터 차단하는 견고한 잠금장치 세트입니다.

---

## 📚 보안 가이드 목차

### 1. [API 속도 제한 (Rate Limiting)](../12-security/rate-limiting.md)

Upstash(Redis)를 사용하여 악성 요청 및 API 오남용을 방지하는 방법을 설명합니다.

### 2. [보안 헤더 (Security Headers)](../12-security/security-headers.md)

브라우저 기반 공격을 차단하기 위한 HTTP 보안 헤더 설정 가이드입니다.

### 3. Supabase RLS (Row Level Security)

데이터베이스 레벨에서 사용자의 데이터를 보호합니다. 모든 테이블에는 기본적으로 본인의 데이터에만 접근할 수 있는 RLS 정책이 적용되어 있습니다.

---

> **Tip**: 처음 시작하실 때는 **보안 헤더** 설정을 먼저 확인하시고, 서비스에 API 노출이 많아지면 **속도 제한** 기능을 활성화하는 것을 추천드립니다. 🚀
